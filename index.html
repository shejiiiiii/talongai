User Manual v3.0

TalongAI

A Raspberry Pi embedded system that uses near-infrared (NIR) spectroscopy to detect eggplant fruit borer infestation — non-destructively and in real time.

SE-CNN · AS7265x · 18-channel · 410–940 nm

What is TalongAI?

Talong is the Filipino word for eggplant. TalongAI is a non-destructive pest detection system — it can tell you whether an eggplant is infested with fruit borer without cutting it open. Two spectral sensors scan the eggplant as a motorized conveyor belt carries it through the detection chamber. A trained machine learning model then classifies each scan segment as Healthy or Infested.

Sensors

The system processes two eggplants simultaneously — one under each sensor — and displays results on a 1024×600 touchscreen dashboard built with Pygame. Results show a final verdict per eggplant based on majority voting across 5 scan segments.

Hardware

All components are connected via I²C through a TCA9548A multiplexer on a Raspberry Pi 4. The conveyor belt is driven by a SparkFun SCMD dual motor controller.

System Architecture

TalongAI runs as two separate OS processes to avoid GPU/OpenGL conflicts between TensorFlow and Pygame on ARM.

ProcessRoleKey Imports
Main ProcessLoads the AI model, drives hardware, runs the scan loopTensorFlow, smbus2, hardware drivers
Display Process (spawned)Renders the Pygame dashboard; reads state from a queuePygame only — no TensorFlow

State updates flow from Main → Display via a multiprocessing.Queue. The display process drains this queue every frame and re-renders. Neither process can see the other's GPU context — this is intentional.

Installation

  1. Clone the repository 
    git clone https://github.com/your-username/talong-ai.git
cd talong-ai
  2. Create a virtual environment (micromamba recommended on Pi) 
    micromamba create -n talong_stable python=3.10
micromamba activate talong_stable
  3. Install dependencies 
    pip install -r requirements.txt
# On Pi ARM, replace tensorflow with:
pip install tensorflow-aarch64
  4. Enable I²C and verify hardware 
    sudo raspi-config      # Interface Options → I2C → Enable
i2cdetect -y 1        # Should show 0x70
  5. Train or provide a model, then place .keras or .pkl in the models/ directory and update MODEL_NAME in config.py.
  6. Run the system 
    python main.py

Booting the Device

When powered on, the system goes through the following stages:

StageDisplay ShowsDuration
Boot screenLogo, progress bar, loading messages~5 s
Hardware initSensor status updates (INIT → READY / NOT CONNECTED)~3 s
Sensor warm-up"WARMING UP" on both panels60 s
Ready"SYSTEM READY" — waiting for scan triggerUntil key press
The 60-second warm-up is required for the AS7265x sensors to reach stable operating temperature. During this time, place the eggplants under the sensor chamber and close the lid.

Power state is persisted in state.json. If the Pi reboots mid-session (e.g. power loss), it automatically relaunches main.py on next boot if the last state was ON.

Auto-start with tmux (optional)

# Start main app
tmux new-session -d -s talongs 'python3 main.py'
# Start macro listener in second session
tmux new-session -d -s talongs_keys 'python3 macro_listener.py'

Scanning Eggplants

  1. Wait for warm-up — both panels must show SYSTEM READY before scanning.
  2. Place eggplants — one under each sensor opening (Sensor 0 = left lane, Sensor 1 = right lane). Close the chamber lid.
  3. Trigger a scan — press 2 on the macro pad, or press Enter on a keyboard.
  4. Watch the scan progress — the conveyor moves through 5 segments. Each panel shows SCANNING while the segment is being read, then MOVING between segments.
  5. Read the result — after all segments, the system processes the votes and displays HEALTHY or INFESTED. The conveyor returns home automatically.
  6. Remove eggplants, then press 2 again to scan the next batch.
⚠ Cancel a scan at any time Press 3 on the macro pad (or type c then Enter on keyboard). The conveyor will return home and the system goes back to READY.

How majority voting works

Each eggplant is scanned across 5 conveyor segments. Each segment produces an independent Healthy or Infested vote. If any segment votes Infested, the entire eggplant is flagged as Infested. This conservative approach minimizes false negatives.

Low-confidence predictions (below CONFIDENCE_THRESHOLD = 60%) are flipped to the opposite label to reduce borderline false positives.

Data Collection

Two scripts handle data collection. Both append rows to the CSV defined as OUTPUT_FILE in config.py.

General collection — collect.py

Use this for both healthy and infested eggplants. You specify which of the 3 conveyor segments face the infested area per rotation.

python scripts/collect.py
  1. Enter H for Healthy or I for Infested.
  2. Enter the Eggplant ID (e.g. H01, I03).
  3. For infested: specify which segments face the damage (e.g. 1 2 or all).
  4. The script runs multiple passes per rotation, rotating the eggplant ~45° between rotations.

Infested-only collection — collect_infested.py

Uses a calibration-then-auto-scan protocol for higher throughput on infested specimens.

python scripts/collect_infested.py
  1. Run 1 (calibration): scan all 3 segments, manually confirm which are infested.
  2. Runs 2–20 (auto): only the confirmed infested segments are scanned automatically.
Protocol tip: Both sensors save data simultaneously (suffix _S0 and _S1), effectively doubling your dataset per scan.

Training a Model

SE-CNN (default)

python scripts/train.py

Trains the Squeeze-and-Excitation 1D CNN on the dataset. Uses grouped train/test split (by eggplant ID), augmentation (noise + scaling), class weights, label smoothing, early stopping, and LR reduction on plateau. Output: models/eggplant_pest_model_v5.keras

Random Forest (alternative)

python scripts/train_rf.py

Trains a 500-tree Random Forest with pairwise channel ratio features (18 → 39 features). No augmentation. Output: models/eggplant_pest_rf_v4.pkl

Evaluation

# 5-fold leave-one-eggplant-out cross-validation
python scripts/evaluate_cnn.py

# SVM vs RF baseline comparison
python scripts/baseline.py

Preprocessing pipeline

StepMethodPurpose
1. SmoothingSavitzky-Golay (window=3, poly=2)Remove sensor noise
2. NormalizationStandard Normal Variate (SNV)Correct for scattering / distance variation
3. Outlier removalZ-score per eggplant group (threshold=2.5σ)Drop anomalous scans
4. Augmentation (CNN only)Gaussian noise + multiplicative scalingRegularize against sensor variation

CNN vs RF Backend

Switch backends by changing one flag in config.py:

# config.py
USE_RF = False   # False = SE-CNN (default)
USE_RF = True    # True  = Random Forest

No code changes needed elsewhere — src/analyzer.py reads this flag and loads the appropriate model.

PropertySE-CNNRandom Forest
Input shape(18, 1) — treats channels as a time series(39,) — 18 channels + 21 ratio features
InterpretabilityLow — attention via SE blocksHigh — feature importance rankings
Training time~5–15 min (200 epochs, early stop)~30 s
AugmentationYes (noise + scale copies)No
Label smoothingYes (0.1)No
⚠ RF preprocessing mismatch: Training applies SNV after adding ratio features; inference applies SNV before. This is a known inconsistency noted in the codebase — unify both when retraining from scratch.

Troubleshooting

Sensors not detected

Run i2cdetect -y 1 and confirm 0x70 is visible. Check MUX port numbers in config.py match the physical wiring. The system degrades gracefully — it will still run if one sensor is missing.

TensorFlow import errors on ARM

Standard TF may not have an ARM wheel. Try: pip install tensorflow-aarch64. Ensure you are in the correct micromamba/venv environment.

Pygame display fails / black screen

Ensure DISPLAY=:0 is exported before running. If running headless, a display server must be active. The variable is set at the top of main.py automatically.

Macro pad not found

Run ls /dev/input/by-id/ to find the device path. Update the path in macro_listener.py if it differs. The script falls back to auto-detecting any keyboard with KEY_A and KEY_B.

Model load failure

Both panels show "MODEL FAIL". The display still runs. Check that the model file path in config.py (MODEL_NAME or RF_MODEL_NAME) is correct and that the file exists in models/.

Conveyor doesn't stop at home

Adjust RETURN_DURATION in config.py. A brief braking pulse is applied at the end of return_home() to prevent coasting. Increase the duration if the belt overshoots.
